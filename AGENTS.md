# AI Learning RPG Project Instructions

This repository is a learning curriculum that teaches practical AI use through RPG-style missions, testing workflows, and small safe labs.

When helping in this repo:

- Preserve the mission structure: Objective, Concept, Hands-on exercise, Failure mode, Debrief, Completion criteria.
- Use synthetic, public, or explicitly approved non-sensitive examples only.
- Do not introduce exercises that require real credentials, real customer data, real surveillance, doxxing, social engineering, or unauthorized system access.
- Mark assumptions clearly instead of turning unknowns into facts.
- Keep edits focused and avoid unrelated refactors.
- For code labs, prefer small observable verification steps such as `node --test`, `python -m unittest`, or Playwright specs.
- Preserve the standalone completion path: mandatory missions must remain completable from `qa_ai_academy.html`; repository runtime labs are optional extensions.
- For curriculum text, keep the espionage-RPG tone, but do not let flavor obscure the learning objective.
- When proposing changes, include the learning reason, the file or section affected, and one verification step.
- For longer work, use the protocol in `qa-ai-academy/AI_SESSION_MEMORY_PROTOCOL.md`: pre-session objective, in-session evidence checkpoints, and post-session handoff notes.
